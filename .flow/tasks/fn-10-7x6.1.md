## Approach
- Follow the existing subcommand pattern in `__main__.py:31-106` (`build_parser()`, handler functions)
- Add `"serve"` to the set on `__main__.py:127` (`known_subcommands`)
- Update module docstring at `__main__.py:1-16` to include `serve` subcommand
- Update `build_parser()` docstring to mention `serve`
- App factory pattern: `create_app()` returns configured `FastAPI` instance
- **Config warning check**: In `create_app()`, check `os.environ.get("GEMINI_API_KEY")`. Store warnings in `app.state.config_warnings: list[str]`. Templates will render these as banners (Task 3).
- Mount `StaticFiles` on the app (not router) after API routes — avoids path conflict gotcha
- Create `.keep` placeholder files in `templates/` and `static/` dirs so they exist in git (StaticFiles defaults to `check_dir=True` and errors if dir is missing)
- `_handle_serve()` imports uvicorn and calls `uvicorn.run(create_app(), host=..., port=..., log_level=..., log_config=None)`. Pass `log_level="debug" if args.verbose else "info"` and `log_config=None` to preserve existing `setup_logging()` config
- Template dir: `Path(__file__).parent / "templates"`, static dir: `Path(__file__).parent / "static"`
- `[tool.setuptools.package-data]` in pyproject.toml: `"cal_ai.web" = ["templates/*.html", "static/*.css", "static/*.js"]`
- Test with `from fastapi.testclient import TestClient` (Starlette's TestClient, backed by httpx)
- **PipelineResult extension** (additive, backward-compatible):
  - Add `memory_actions: list[MemoryAction] = field(default_factory=list)` — populated from `write_result.actions` in Stage 4
  - Add `extraction_usage_metadata: list = field(default_factory=list)` — populated from `extraction.usage_metadata` in Stage 2
  - Both default to empty lists so all existing code is unaffected

## Key context
- StaticFiles must be mounted on `app` object, not an `APIRouter`, and after API routes are included
- StaticFiles defaults to `check_dir=True` — directory must exist or it errors on startup
- `from fastapi.testclient import TestClient` is the correct import (NOT `from httpx`)
- `uvicorn.run()` with `log_config=None` prevents uvicorn from applying its own logging config
- The `monkeypatch_env` and `clean_env` fixtures in `tests/conftest.py` handle env var isolation
- `MemoryAction` is imported from `cal_ai.memory.models` — already a Pydantic model
- `ExtractionResult.usage_metadata` contains Gemini SDK `usage_metadata` objects (see `llm.py:32-47`)
- Pipeline Stage 4 is skipped in dry-run mode — `memory_actions` stays empty, which is correct behavior
